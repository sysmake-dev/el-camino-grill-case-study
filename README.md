# El Camino Grill -- Case Study

**Type:** Concept / Spec Project  
**Client:** Local family-owned food truck  
**Role:** Design | Frontend Development | Information Architecture

## OVERVIEW

El Camino Grill is a family-owned food truck serving fresh, affordable street food to their local community.

This case study explores the design and development of a simple, mobile-first website intended to help customers quickly access essential information such as the menu, hours, and location.

The case study focuses on clarity, performance, and usability under real-world constraints.

## PROBLEM

The business relied primarily on word-of-mouth and social media posts, making it difficult for customers to find consistent and up-to-date information.

Key challenges included:

- No centralized location for menu and pricing
- Inconsistent location and hours visibility
- Poor readability on mobile devices
- Dependence on third-party platforms

## CONSTRAINTS

The project was designed with realistic constraints in mind:

- Mobile-first usage
- Limited technical resources
- Fast load times
- Simple content updates
- Long-term maintainability

## DESIGN APPROACH

The design process began with low-fidelity sketches to establish layout and information hierarchy.

Priorities included:

- High contrast for outdoor readability
- Minimal navigation
- Large tap targets
- Clear visual hierarchy

Hand-drawn sketches and digital mockups are included as part of the case study.

## DEVELOPMENT APPROACH

The site is built using semantic HTML and lightweight CSS, with an emphasis on:

- Accessibility
- Performance
- Maintainability
- Simplicity

No external frameworks required.

## OUTCOME

The final result is a clean, mobile-optimized website that allows customers to:

- View the menu
- Check current hours
- Find the truck’s location
- Get in touch easily

The site reduces friction for customers while giving the business full control over their online presence.
